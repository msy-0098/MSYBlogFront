# ICP 备案信息页脚展示设计

- 日期：2026-07-20
- 范围：前端公共页脚
- 状态：已获用户确认

## 1. 背景

博客域名 `masenyu.top` 已完成 ICP 备案，需要在公开站点页脚展示备案号，方便访客核验并满足网站信息公示需求。当前页脚已有 RSS、Sitemap 和技术栈信息，但部分中文文案存在乱码，需要一并修复。

## 2. 目标

1. 在公共页面页脚展示服务备案号 `豫ICP备2026032113号-1`。
2. 将备案号设置为指向工信部备案查询入口的外链，并在新窗口打开。
3. 保持后台管理路由不显示公共页脚的现有行为。
4. 修复页脚中现有的中文乱码，不扩大本次任务范围。
5. 保持桌面端与移动端的现有响应式布局。

## 3. 非目标

- 不新增后端接口或数据库字段。
- 不修改备案主体信息、域名解析或服务器配置。
- 不在仓库中写入个人令牌、服务器密码、环境变量或其他敏感信息。
- 不调整页脚整体视觉风格和布局体系。

## 4. 方案

只修改 `src/components/layout/AppFooter.vue`：

- 使用现有的 `currentYear` 动态展示版权年份。
- 将版权文案恢复为 `© {currentYear} 马森雨`。
- 将简介文案恢复为 `写作、项目与持续成长的公开记录`。
- 在页脚链接区域加入备案链接：
  - 文案：`豫ICP备2026032113号-1`
  - 链接：工信部备案查询入口
  - 属性：`target="_blank"`、`rel="noreferrer"`
- 使用现有 `.footer-links` 样式，不新增 CSS；备案号在现有 flex 布局下自然换行。

## 5. 验证

在 `frontend/` 仓库内执行：

1. `pnpm test`
2. `pnpm build`
3. `git diff --check`
4. `git status --short`，确认只提交本次设计文档和实现相关文件，不包含 `node_modules/`、`dist/`、缓存、日志或敏感配置。

实现完成后，按项目规则在 `frontend/` 仓库单独提交，并分别推送 GitHub `origin` 与 Gitee `gitee` 当前分支。
