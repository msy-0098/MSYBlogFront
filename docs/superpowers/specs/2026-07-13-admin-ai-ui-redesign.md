# 管理端 UI 与独立 AI 工作区重构设计

日期：2026-07-13  
状态：用户已批准  
适用仓库：frontend

## 1. 背景

当前管理端样式大量集中在 `global.css`，颜色、圆角和控件体系不统一；固定侧栏、顶栏和 dashboard 网格缺少中间尺寸响应式。文章编辑器混用原生控件与 Element Plus。工作台使用整块 `v-loading`，AI 助手只是 textarea、按钮和最终 alert，不能连续聊天或管理历史。

## 2. 已批准方向

- 采用分层渐进式改造，不更换 Vue 3、TypeScript、Pinia 和 Element Plus；
- 视觉选择“清透专业”：浅灰蓝背景、白色表面、低对比边框、克制阴影、蓝色强调和充足留白；
- 新增独立 `/admin/ai` 工作区；
- 聊天历史由服务器 PostgreSQL 保存；
- 使用 SSE 增量渲染；
- dashboard 普通内容立即显示，AI 洞察局部异步加载；
- 会话支持新建、切换、自动命名、手动重命名、删除单个和清空全部；
- 首轮不包含搜索、收藏和导出。

## 3. 管理端视觉系统

新增管理端语义令牌，覆盖：

- 页面、卡片和弱化表面；
- 主、副和弱文本；
- 边框、主色、危险色和状态色；
- 小、中、大圆角；
- 小、中阴影；
- 侧栏宽度、顶栏高度和内容最大宽度；
- Element Plus 主题变量和默认密度。

将后台样式从 `global.css` 拆到独立 `admin.css`。前台设计令牌和游客端样式保持稳定。统一后台页面标题、工具栏、统计卡片、内容面板、表格面板、表单分区、空状态和危险操作区。

## 4. AdminLayout

### 4.1 导航分组

- 概览：工作台；
- 内容：文章、分类、标签、项目；
- 互动：评论、用户；
- 智能工具：AI 助手；
- 系统：访问安全、站点设置。

移除不可操作的假搜索框。若后续实现真实全局搜索，再以独立功能加入。

### 4.2 响应式

- 桌面：完整侧栏和顶栏；
- 中等屏幕：侧栏收窄为图标导航，内容网格降列；
- 手机：侧栏进入 Drawer，顶栏仅保留菜单、当前页面和用户操作；
- 表格使用统一横向滚动容器或按优先级隐藏次要列；
- 交互点击目标不小于适合触控的尺寸；
- 不允许导航、顶栏或表格遮挡内容。

## 5. 页面统一

现有 CRUD 和路由保持不变，主要统一：

- 页面标题和中文 kicker；
- 筛选区与操作区；
- 卡片层级；
- 表格边框、表头、行高和 hover；
- 表单栅格、错误提示和按钮顺序；
- 空状态、错误状态、局部刷新状态；
- Element Plus 与文章编辑器控件视觉。

文章编辑器保留 Markdown 能力，但按钮、选择器、上传入口和可替换输入控件统一到 Element Plus 风格，不继续维护两套视觉体系。

## 6. 独立 AI 工作区

新增管理路由：

```text
/admin/ai
```

桌面采用三段式结构：

```text
管理端主导航 | 会话列表 | 聊天区域
```

### 6.1 会话列表

- 新建会话；
- 当前会话高亮；
- 标题、消息数量和最后时间；
- 手动重命名；
- 删除单个会话；
- 清空全部记录；
- 删除和清空使用二次确认；
- 首轮不展示尚未实现的搜索和收藏控件。

### 6.2 聊天区域

- 用户和 AI 消息使用不同气泡与对齐方式；
- AI 消息支持逐段追加；
- 保留换行和代码内容的可读性；
- 自动滚动仅在用户位于底部附近时执行，用户向上阅读时不得强制拉回；
- 显示 streaming、completed、aborted 和 failed 状态；
- 失败消息提供重试入口；
- 生成中按钮切换为“停止生成”；
- 输入框支持 Enter 发送、Shift+Enter 换行；
- 发送时立即显示用户消息，不等待服务器完整响应。

### 6.3 状态管理

抽离 AI 会话 store/composable，负责：

- 会话列表与当前会话；
- 消息加载；
- SSE 生命周期；
- AbortController；
- 局部错误；
- 新建、重命名、删除和清空；
- 路由切换和组件卸载时取消未完成请求。

PostgreSQL 是事实来源，前端不把 localStorage 作为正式会话存储。可保留当前会话 ID 等无敏感的界面偏好，但刷新后必须从服务端重新读取正式数据。

## 7. SSE 客户端

由于浏览器原生 `EventSource` 不便发送带 JSON body 的 POST，请求使用 `fetch` 和 `ReadableStream`：

1. 带管理员 JWT 发起 POST；
2. 校验 HTTP 状态和 `text/event-stream`；
3. 按 SSE 空行边界解析事件；
4. `meta` 建立消息元数据；
5. `delta` 追加到当前 AI 消息；
6. `done` 标记完成并刷新会话摘要；
7. `error` 显示稳定错误信息；
8. 用户停止或组件卸载时调用 AbortController。

解析器独立为纯函数或独立模块，以便测试分包、半包、多事件、UTF-8 字符、错误事件和连接中断。

## 8. Dashboard 非阻塞体验

普通 dashboard 数据和 AI 洞察分开请求：

- 普通统计使用卡片骨架，不遮挡整个页面；
- 数据返回后立即渲染；
- AI 洞察卡片独立请求；
- 刷新 AI 洞察时保留旧结果并显示轻量刷新状态；
- AI 失败仅影响该卡片，显示重试；
- 工作台不再包含完整聊天 textarea，只提供进入 AI 工作区的快捷入口和最近会话摘要。

其他页面也遵守：首次加载用局部骨架，保存用按钮级 loading，刷新时优先保留旧数据，禁止整页阻塞遮罩。

## 9. 可访问性与交互

- 所有图标按钮提供可读标签；
- Drawer、确认弹窗和菜单支持键盘操作与焦点管理；
- 流式状态不仅依赖颜色；
- 危险操作使用明确文案和二次确认；
- 支持 `prefers-reduced-motion`；
- 颜色对比满足清晰阅读；
- 手机端输入区域不被软键盘或底部安全区遮挡。

## 10. 前端 CI

复用现有 pnpm workflow，并补充：

- 管理端路由和导航分组测试；
- AI 会话 store 测试；
- SSE 分片解析测试；
- 停止生成和组件卸载取消测试；
- 会话重命名、删除与清空确认测试；
- dashboard 普通数据不等待 AI 的测试；
- 响应式关键 class 和状态渲染测试；
- `pnpm test` 与 `pnpm build` 继续作为部署门禁。

## 11. 验收标准

- 管理端视觉统一，颜色、卡片、按钮、表单、表格和文案不再割裂；
- 桌面、平板和手机均无导航堆叠、遮挡和横向溢出；
- 打开 dashboard 时没有整页 loading；
- DeepSeek 较慢或失败时，其他管理功能可立即使用；
- AI 页面可新建、切换、重命名、删除和清空会话；
- 刷新和换设备后可恢复服务端历史；
- AI 文本边到达边显示；
- 用户可停止生成，已生成内容保留；
- 所有新增测试通过，`pnpm build` 成功。
