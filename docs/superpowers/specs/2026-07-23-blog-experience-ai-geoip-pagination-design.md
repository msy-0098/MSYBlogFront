# 博客体验、AI 能力、IP 查询与分页设计（前端）

- 日期：2026-07-23
- 状态：已确认，待实施
- 配套后端文档：`backend/docs/superpowers/specs/2026-07-23-blog-experience-ai-geoip-pagination-design.md`
- 交付方式：四阶段开发，每阶段独立验证、提交并推送，全部完成后统一部署

## 1. 背景与目标

当前项目已经具备验证码发送、基础 Axios 错误拦截、AI 工作台和部分列表能力，但仍存在刷新后倒计时丢失、错误状态码直接展示、邮件缺少品牌视觉、AI 原生 `fetch` 错误未统一、长列表不可完整浏览等问题。

本次前端目标如下：

1. 提供稳定、可恢复的验证码倒计时体验；
2. 统一游客端、管理端及 AI SSE 请求的友好错误提示；
3. 配合后端发送带 MSY 品牌 Logo 的验证码邮件；
4. 将管理端 AI 补齐为博客创作与运营助手；
5. 在“访问安全”页面增加管理员 IP 归属地查询；
6. 为真正需要分页的列表补齐分页或增量加载。

## 2. 非目标

本期不包含：

- 游客端 AI 聊天入口；
- 复杂多供应商管理页面；
- RAG、向量检索或知识库；
- 对外开放的 IP 查询页面；
- 对 Dashboard Top 5、Top 10 等固定摘要强行分页；
- 保存管理员的 IP 查询历史。

## 3. 验证码倒计时

### 3.1 状态模型

倒计时按“标准化邮箱 + 用途”隔离：

```ts
interface EmailCodeCooldown {
  email: string
  purpose: 'register' | 'reset'
  expiresAt: number
}
```

使用 `sessionStorage` 保存绝对过期时间，而不是仅保存剩余秒数。页面刷新、弹窗关闭再打开或同一标签页路由切换后，均通过 `expiresAt - Date.now()` 恢复状态。

建议键名：`email-code-cooldown:<purpose>:<normalized-email>`。

### 3.2 交互规则

- 点击发送前校验邮箱格式；
- 发送成功后采用后端 `cooldownSeconds` 启动 60 秒倒计时；
- 后端返回 429 时读取 `retryAfter`，同步倒计时并显示友好提示；
- 网络失败或发送失败时不启动新倒计时；
- 邮箱或用途改变时切换到对应的独立倒计时；
- 倒计时结束后清理 `sessionStorage`；
- 按钮文案使用“重新发送（59s）”，禁用期间保持可读且不引起布局跳动；
- 组件卸载时清理定时器，避免重复计时和内存泄漏。

## 4. 全站友好错误提示

### 4.1 统一错误模型

公共 Axios、管理端 Axios、AI SSE 的原生 `fetch` 以及页面级请求都映射为统一错误：

```ts
interface FriendlyApiError extends Error {
  status?: number
  code?: number
  retryAfter?: number
  kind: 'validation' | 'auth' | 'permission' | 'not-found' | 'conflict' |
        'rate-limit' | 'network' | 'timeout' | 'server' | 'unknown'
}
```

### 4.2 映射规则

| 场景 | 默认用户提示 |
| --- | --- |
| 400 | 提交内容有误，请检查后重试 |
| 游客端 401 | 登录状态已失效，请重新登录 |
| 管理端 401 | 登录已过期，请重新登录 |
| 403 | 暂无权限执行此操作 |
| 404 | 请求的内容不存在或已被移除 |
| 408 / 超时 | 请求超时，请稍后重试 |
| 409 | 当前数据已发生变化，请刷新后重试 |
| 422 | 提交内容未通过校验，请检查后重试 |
| 429 | 操作过于频繁，请稍后再试 |
| 500 及以上 | 服务暂时不可用，请稍后再试 |
| 网络离线 | 网络连接异常，请检查网络后重试 |
| 未知错误 | 操作失败，请稍后重试 |

可信业务提示可以保留，例如“验证码错误或已过期”“邮箱或密码错误”。以下内容不得直接展示：HTTP 状态码、`AxiosError` 英文文本、SQL、SMTP、AI Provider 原始错误、堆栈或内部配置。

### 4.3 UI 行为

- 表单字段错误优先显示在字段附近；
- 页面操作使用 Element Plus 消息或通知；
- 管理端 401 保留清理会话并跳转登录页的行为；
- AI SSE 解析非 2xx 响应体后也必须经过同一映射器；
- 停止生成属于用户主动操作，不弹失败提示；
- 移除可被统一消息组件替代的 `alert()`。

## 5. 品牌邮件的前端配合

用户提供的 MSY Logo 将由后端裁剪多余黑色留白、压缩为邮件专用 PNG，并通过 CID 嵌入邮件。前端不上传或动态传递 Logo，只负责保持验证码发送成功、冷却时间和错误提示的一致体验。

邮件设计要求：

- 黑底银色横向品牌图，视觉宽度约 180–220px；
- 验证码是邮件主体的最高视觉层级；
- 明确有效期和安全提醒；
- 提供图片替代文字与纯文本备用正文；
- 手机端可读，不依赖外部图片加载。

## 6. 管理端 AI：博客创作与运营助手

### 6.1 产品定位

AI 工作台服务于管理员，不面向游客。核心场景包括文章构思、标题与摘要、正文润色、SEO 建议、运营分析和异常洞察。

### 6.2 状态与健康检查

设置页展示：

- AI 是否启用；
- Provider 类型与脱敏配置摘要；
- 模型名称；
- 配置完整性；
- 最近健康检查结果、耗时和时间；
- 手动“测试连接”按钮。

前端永不回显完整 API Key。

### 6.3 AI 工作台

- 会话列表采用游标分页，每次 20 条；
- 消息历史采用反向游标分页，每次 50 条，可向上加载更早消息；
- Markdown 渲染支持代码高亮；
- 支持复制整段回答和单个代码块；
- 支持停止生成、失败重试与明确状态；
- 重试基于原问题和原上下文创建新尝试，不污染上下文；
- `aborted` 与 `failed` 均不可作为后续完整上下文；
- 页面离开或主动停止时正确中止 SSE 请求；
- 输入区展示长度限制，超限时阻止提交；
- 429 时展示可理解提示和可用的等待时间。

### 6.4 创作 Prompt 模板

提供结构化模板，至少覆盖：

1. 文章选题与大纲；
2. 标题、摘要和关键词；
3. 正文扩写或精简；
4. 语气与可读性润色；
5. SEO 检查；
6. 评论与运营总结。

模板只负责填充输入，不自动发送，管理员可编辑后确认。

### 6.5 文章润色

原有直接覆盖编辑器的流程改为：

1. 选择润色目标或输入要求；
2. 生成候选内容；
3. 展示原文与候选内容的差异预览；
4. 管理员确认后应用；
5. 应用后提供一次撤销；
6. 失败时保留编辑器原文。

### 6.6 运营分析报告

- 支持 7、30、90 天时间范围；
- 分析访问趋势、热门文章、站内搜索、评论、异常 IP 和内容建议；
- 以 Markdown 展示；
- 保存最近 20 份；
- 支持查看、删除和重新生成；
- 展示模型、Token、耗时、状态和生成时间；
- 不新增访客原始隐私数据采集。

## 7. IP 归属地查询

入口集成到现有“访问安全”页面，仅管理员可见。

功能包括：

- 输入 IPv4 或 IPv6，按 Enter 或按钮查询；
- 展示国家、省份、城市、经纬度、ASN、运营商或组织、数据源和数据库更新时间；
- Top IP 和封禁记录提供“一键查询”；
- 私网、回环、链路本地、组播、未指定和保留地址显示明确不可查询原因；
- 查询失败不触发自动封禁；
- 不保存查询历史；
- 经纬度仅展示数据，不引入地图依赖。

## 8. 分页方案

### 8.1 游客端

| 列表 | 方式 | 每次数量 | 交互 |
| --- | --- | ---: | --- |
| 评论 | `created_at + id` 游标 | 20 | 加载更多 |
| 归档 | `published_at + id` 游标 | 50 | 加载更多 |
| 首页项目 | `limit` 参数 | 3 | 固定展示 |

项目页、友链页和分类入口属于低数量人工维护集合，不做客户端分页，但服务端必须限制最大返回量。

### 8.2 管理端

| 列表 | 方式 | 默认每页 |
| --- | --- | ---: |
| 用户 | 页码分页 | 20 |
| 分类 | 页码分页 | 20 |
| 标签 | 页码分页 | 20 |
| 项目 | 页码分页 | 20 |
| 友链 | 页码分页 | 20 |
| IP 封禁记录 | 页码分页 | 20 |
| AI 会话 | `updated_at + id` 游标 | 20 |
| AI 消息 | `sequence` 反向游标 | 50 |

页码分页统一支持页码、每页数量、总数和必要筛选；删除当前页最后一条后自动回退到有效页。Dashboard Top 列表和趋势摘要维持固定数量。
