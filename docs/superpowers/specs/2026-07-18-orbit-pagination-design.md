# 分页轨道视觉统一设计

- 日期：2026-07-18
- 范围：前台文章列表、分类/标签列表、搜索结果，以及后台文章/评论管理分页
- 目标：把当前浏览器默认风格的分页按钮替换成与个人博客品牌一致的“轨道页码”交互

## 1. 背景

当前前台分页由两个普通 `<button>` 和一个文本页码组成，视觉上像浏览器默认控件；后台分页使用 Element Plus 默认样式。用户希望分页更有创意，同时保持文章阅读页面的清爽、轻盈和科技感。

## 2. 设计方向

采用“轨道页码（Orbit Pagination）”：

- 当前页显示为蓝色胶囊，文案为 `第 X / Y 页`；
- 当前页周围使用低对比度虚线轨道和少量圆点作装饰；
- 两侧使用带方向箭头的“上一篇 / 下一篇”按钮；
- 轨道只承担装饰，不覆盖按钮点击区域，不影响文本可读性；
- 在窄屏下自动折叠为上下两行，保证按钮有足够点击面积；
- 深色模式下沿用项目 token，只调整边框、阴影和轨道对比度。

## 3. 组件与数据流

新增 `src/components/common/OrbitPagination.vue`，组件只负责分页展示和发出目标页事件：

- Props：`currentPage`、`totalPages`、可选 `label`；
- Emits：`change(page: number)`；
- 当只有一页时不渲染；
- 第一页禁用“上一篇”，最后一页禁用“下一篇”；
- 点击当前页不触发重复请求；
- 使用 `aria-label`、`aria-current`、`aria-disabled`，支持键盘操作；
- 通过 `data-test="orbit-pagination"`、`data-test="orbit-pagination-prev"`、`data-test="orbit-pagination-next"` 暴露稳定测试入口。

数据请求仍由页面负责，保持当前 API 和 `currentPage` 状态逻辑不变：

- `PostListView.vue`：文章、分类、标签共用；
- `SearchView.vue`：搜索分页共用；
- `AdminPostsView.vue`、`AdminCommentsView.vue`：替换当前 Element Plus 分页，保留现有查询参数和后台表格数据刷新逻辑。

## 4. 视觉与交互细节

- 默认颜色使用项目品牌蓝色 token，不引入新的渐变色体系；
- 当前页胶囊有轻微蓝色光晕，hover 时轨道圆点做短促位移；
- 按钮 hover 时上移 1px，active 时回落，避免夸张动效；
- `prefers-reduced-motion: reduce` 时关闭轨道和按钮动画；
- 禁用态降低透明度并禁止 pointer interaction；
- 组件使用 `button`，不使用不可访问的 `div` 点击；
- 移动端按钮文案可缩短为 `上一页` / `下一页`，当前页仍保留页数信息。

## 5. 测试策略

先增加失败测试，再实现组件：

1. 组件测试：一页时隐藏，多页时渲染；
2. 组件测试：第一页/最后一页的禁用态；
3. 组件测试：点击上一页、下一页发出正确页码；
4. 组件测试：当前页标记和 ARIA 属性；
5. 页面测试：文章列表和搜索页使用统一组件，并继续传递正确的分页参数；
6. 运行前端全量单元测试和生产构建；
7. 通过浏览器检查桌面、移动窄屏和深色模式。

## 6. 验收标准

- 前台文章、分类、标签、搜索分页全部显示轨道页码样式；
- 后台文章、评论分页也完成统一视觉，不改变分页查询行为；
- 首尾页按钮状态正确，点击不会发出越界请求；
- 键盘可操作，屏幕阅读器可识别当前页和按钮状态；
- 不出现横向滚动、按钮遮挡或布局跳动；
- `pnpm test` 和 `pnpm build` 通过；
- 本次改动只提交前端相关文件，不提交依赖、构建产物、`.superpowers` 临时 mockup 或敏感配置。

## 7. 非目标

- 不修改后端分页 API；
- 不改变每页文章数量；
- 不新增数字页码跳转、分页缓存或预加载策略；
- 不重做后台表格布局和其他导航控件。
